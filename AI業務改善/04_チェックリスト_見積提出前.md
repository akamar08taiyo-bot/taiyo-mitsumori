# 資産②：見積提出前チェックリスト（印刷して手元に）

**用途**: 印刷・PDF・共有ボタンを押す**前**の30秒確認。出戻りと事故をゼロにする。
**根拠**: アプリが警告する項目（宛名空欄・発行日未設定・商品0件・合計¥0）＋実務で事故になりやすい項目で構成。

---

## A. 宛先・日付（間違うと信用に関わる）

- [ ] 宛先（施設名）が入っている。**施設名の漢字が正しい**（苑/園、ヶ/が）
- [ ] 敬称（御中／様）が宛先に合っている
- [ ] 見積日が今日（または意図した日付）になっている
- [ ] 見積有効期限・支払条件が案件に合っている（月末締め翌月末払いのままでよいか）

## B. 商品・金額（間違うと利益に関わる)

- [ ] 商品名に規格・入数が入っている（後から「どのサイズ？」と聞かれない状態）
- [ ] 数量×単価＝金額をざっと暗算で1行検算した
- [ ] 税区分（税抜／税込／非課税）が正しい。**下部の備考文と合計表示が連動しているか見た**
- [ ] 値引き・特価の行は意図どおり（赤字表示・マーカーの付け忘れ/付けすぎ）
- [ ] 「入力途中（未追加）の商品」の警告が出たら、追加してから印刷した

## C. 見せ方（客先に出る紙の品質）

- [ ] **社内確認用印刷と客先用を取り違えていない**（社内用には仕入単価・利益率が載る。絶対に客先に出さない）
- [ ] 印刷非表示にした行が、印刷プレビューで本当に消えている
- [ ] 価格改定資料のとき：値上がり率の列を「出す/出さない」を意図して選んだ
- [ ] プレビューで1ページに収まり方を確認した（表がはみ出していない）

## D. 保存・送信（押す前の最後の指差し)

- [ ] 保存した（フォルダ＝施設名、名前＝「YYYY-MM 用途」。例：`2026-07 通常品`）
- [ ] **上書き保存のとき：営業所共有の元データを更新していないか確認した**（共有データ編集中バッジが出ていないか）
- [ ] メール添付するファイルが、いま作ったその見積である（ダウンロードフォルダの古いPDFを添付していない）
- [ ] 送信先メールアドレスが宛先の施設と一致している

---

**運用**: 印刷して机に貼る。新しく入った人には最初にこれを渡す。
**更新**: 事故・ヒヤリが起きたら、その日のうちに1行足す（このファイルに追記していくのが「業務OS」）。
