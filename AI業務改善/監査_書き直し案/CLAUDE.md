# taiyo-mitsumori — 見積書・提案書作成システム

太陽シルバーサービス㈱の社内向けWebアプリ。単一HTMLで動作し、GitHub Pagesで配信する。
利用者は全19営業所の担当者。壊れると全営業所の見積業務に影響する。慎重に変更すること。

## ファイルの役割（最重要）

- `index.html` — **ビルド済み成果物**（minify済み・依存同梱・約1.6MB）。これが本番配信物。
- 開発用ソース（JSX入りHTML）は**このリポジトリの外**にある：`build.js` が `../index.html` を読む。
  ソースが手元に無いセッションでは、大規模改修を始める前に必ずユーザーへソースの所在を確認すること。
- `見積書作成.txt` — 2026-06-23時点の旧ソースのスナップショット。**現行仕様ではない**
  （価格改定モードを含まない）。仕様の根拠に使わないこと。
- `DESIGN.md`（→ `.claude/skills/mitsumori-design/SKILL.md`）— UI・印刷帳票のデザイン規約。
  画面や帳票を触るときは必ず先に読む。

## ビルドと確認

- ビルド: `npm install` 後、`node build.js <ソースhtml> <出力先>`（既定は `../index.html` → `./index.html`）
- ローカル確認: `npx http-server . -p 5599 -c-1`（`.claude/launch.json` と同じ）
- 変更後は SKILL.md 末尾の品質チェックリストで確認する（印刷はA4プレビュー実寸で見る）。

## 変更時のルール

1. `index.html` の**手編集は最小限**（文言修正など）。ロジック改修はソース側で行い再ビルドが原則。
2. 保存系・共有系（localStorage / Firebase同期）のキーとデータ形式を変えるときは、
   既存ユーザーのデータが読めなくなる互換性影響を必ず先に説明し、承認を得る。
3. 破壊的操作（データ形式変更・機能削除・Firebase設定変更）は勝手に行わない。
4. コミットメッセージは日本語で「何をどう変えたか」を1行（既存の履歴の流儀に合わせる）。

## 変わる事実（更新が必要になったらここも直す）

- 営業所マスタ（住所・電話・所長名）はソース内 `OFFICE_MASTER` に直書き。
  **人事異動・移転の際はここを更新して再ビルド**が必要（アプリ内「営業所設定」は端末ごとの上書きに過ぎない）。
- Firebase プロジェクト: taiyo-mitsumori-03724（匿名認証＋Firestore同期）。

## 用語

- 仕切り＝メーカー卸価格（PDF/画像で管理）。施設＝顧客（病院・介護施設）。
- モードは3つ: 見積作成（数量×単価）／提案書作成（ケース入数×単価・削減提案）／価格改定（改定前後比較）。
